# 管理后台 UI 重设计

**日期：** 2026-05-19  
**状态：** 已批准

## 背景

现有管理后台使用 Element Plus 默认样式（深蓝侧边栏 + 浅灰主区），视觉上较为简陋，与公开博客的精心设计形成落差。本次重设计目标是将管理后台升级为 Stripe Dashboard 风格的专业浅色管理界面，提升整体视觉品质和使用体验。

## 设计方向

**风格：** Stripe Dashboard 精致商务风  
**核心特征：**
- 白色卡片 + 浅蓝灰页面背景（`#f6f9fc`）
- 极细边框（`#e3e8ef`），无阴影或极轻阴影
- 靛紫强调色（`#635bff`）用于按钮、选中态、focus 光晕
- 表格无竖线，仅横向分隔，hover 行背景微变
- 状态用彩色小圆点 + 文字徽章
- 操作按钮用细边框 outline 样式，hover 时填充浅色背景

## 配色系统

| 用途 | 颜色值 |

## 布局结构

```
┌─────────────────────────────────────────────────────────┐
│  侧边栏 220px（可拖拽 100-400px）                        │
│  白底 + 右边框 #e3e8ef                                   │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  顶栏 56px（sticky）白底 + 下边框                   │ │
│  │  左：页面标题  右：操作按钮                          │ │
│  ├─────────────────────────────────────────────────────┤ │
│  │  内容区  padding: 24px 28px                         │ │
│  │  RouterView                                         │ │
│  └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**侧边栏结构：**
1. Logo 区（28px 图标 + 博客管理 + Admin Dashboard 副标题）
2. 导航分组，分组标签用 11px 大写灰色
   - 概览：仪表盘
   - 内容：文章管理、分类管理、标签管理、评论管理（待审数量徽章）
3. 底部用户信息行（头像 + 用户名 + 角色 + 退出按钮）

**顶栏：** 每个页面独立渲染页面标题 + 右侧操作按钮（如"新建文章"）

## 各页面设计

### AdminLayout

- 侧边栏：白底，右边框，可拖拽（保留现有拖拽逻辑）
- 导航选中态：`#f0effe` 背景 + `#635bff` 文字，左侧无竖条
- 顶栏：sticky，白底，下边框，高度 56px
- 主区背景：`#f6f9fc`

### LoginView

- 全屏居中，背景 `#f6f9fc`
- 白色卡片，圆角 10px，细边框，轻阴影（`0 4px 24px rgba(0,0,0,0.08)`）
- 卡片内：Logo 图标（28px 靛紫方块）+ 标题"博客管理后台" + 副标题
- 表单：用户名、密码（带显示/隐藏切换），label 在上方
- 输入框 focus：`#635bff` 边框 + `rgba(99,91,255,0.1)` 光晕
- 登录按钮：全宽，`#635bff` 背景，圆角 6px

### DashboardView

- 顶部 3 列统计卡片（文章总数 / 待审评论 / 本月浏览）
  - 白底卡片，细边框，无阴影
  - 标签：12px 大写灰色；数值：28px 粗体；变化：12px 绿/红
- 下方文章列表卡片（最近 5 篇，带搜索框）

### PostsView

- 顶部工具栏：搜索框（左）+ "显示已删除"checkbox + "新建文章"按钮（右）
- 表格：`el-table` 保留，覆盖样式
  - 表头：12.5px 大写灰色，`#f6f9fc` 背景
  - 行 hover：`#f9fbfd`
  - 状态列：彩色徽章（已发布绿 / 草稿橙 / 已删除灰）
  - 操作列：outline 小按钮（编辑靛紫 / 删除红 / 恢复绿）
- 分页：`el-pagination`，覆盖为靛紫配色

### PostEditView

- 顶栏：面包屑（文章管理 / 新建 or 编辑）+ 取消/保存按钮
- 左侧主区：标题、Slug、摘要输入框 + `md-editor-v3`（保持不动）
- 右侧侧边栏（可拖拽，保留现有逻辑）：
  - "发布设置"区块：状态开关
  - "分类标签"区块：分类下拉、标签多选、封面图 URL
  - 区块间细线分隔，白色卡片包裹

### CategoriesView / TagsView

- 与 PostsView 同样的表格风格
- 新建/编辑弹窗（`el-dialog`）：
  - 白底，圆角 8px，细边框
  - 表单 label 在上方，输入框 focus 靛紫光晕
  - 按钮：取消（outline）/ 确认（靛紫填充）

### CommentsView

- 顶部筛选：三个 Tab 按钮（全部 / 待审核 / 已通过），选中态靛紫底色白字
- 表格：同 PostsView 风格
  - 评论内容列：单行截断，hover 展示完整内容（`el-tooltip`）
  - 操作：通过（绿色 outline）/ 删除（红色 outline）

## Element Plus 覆盖策略

保留的 Element Plus 组件：`el-table`、`el-dialog`、`el-form`、`el-input`、`el-select`、`el-pagination`、`el-tooltip`、`el-popconfirm`、`md-editor-v3`

通过 CSS 变量覆盖 Element Plus 主题：
```css
--el-color-primary: #635bff;
--el-border-color: #e3e8ef;
--el-bg-color: #ffffff;
--el-fill-color-light: #f6f9fc;
--el-border-radius-base: 6px;
```

替换为 Tailwind 自定义的组件：按钮、徽章、状态标签、卡片容器、顶栏、侧边栏

## 验证方式

1. 启动前端开发服务器：`cd frontend && npm run dev`
2. 访问 `http://localhost:6006/admin/login`，验证登录页样式
3. 登录后验证仪表盘、文章列表、分类、标签、评论各页面
4. 验证新建/编辑文章流程（PostEditView 拖拽侧边栏、md-editor-v3 正常工作）
5. 验证弹窗（新建分类/标签）样式
6. 验证响应式：侧边栏拖拽功能保持正常
7. 运行 `npm run build` 确认无编译错误
